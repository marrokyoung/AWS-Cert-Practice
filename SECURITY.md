# Security Policy

## Supported Versions

This project is in early development and does not yet publish stable versioned releases.

Security fixes are applied to the current `main` branch.

## Reporting a Vulnerability

Please do not open public GitHub issues for security problems.

Preferred reporting path:

1. Use GitHub's private vulnerability reporting for this repository if the "Report a vulnerability" option is available.
2. If private reporting is not available, contact [@marrokyoung](https://github.com/marrokyoung) directly on GitHub and keep the details non-public until the issue is reviewed.

When reporting, include:

- A short description of the issue
- Impact and affected area
- Steps to reproduce
- Any proof-of-concept or logs that help validate the report
- Suggested remediation, if you have one

## Response Expectations

- Initial acknowledgement target: within 7 days
- Status update target: within 14 days when the report is actionable

## Scope

This policy covers:

- The web application
- The Rust API
- CI/CD and repository automation
- Content-processing scripts and build pipeline

Out-of-scope items may still be reviewed, but they may be handled as normal bugs instead of security issues.
